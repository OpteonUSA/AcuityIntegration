The Non-Obvious Complexity

The technical code work is the easy part. Each tier from 3 onward has a business workflow design problem attached that must be answered before the code can be designed:

  • Tier 4 (revisions): If Valor sends a revision request, who in Opteon sees it? In JaroDesk? In a Power App? In a Teams channel? Until ops answers this, the technical wiring can't be designed.
  • Tier 5 (fee approval): Who has authority to approve a $50 fee bump? A $500 fee bump? Are there auto-approve thresholds?
  • Tier 6 (due date negotiation): Does Opteon push back when Valor proposes a date, or always accept?
  • Tier 8 (UAD compliance): UAD findings probably need to roll up to underwriting / lender; who owns that mapping?
Plan accordingly: These workflow design conversations historically take longer than the code itself. Budget 1–2 weeks of meeting cycles per tier just for design alignment, separate from build time.

Recommended Phased Delivery

Concrete phases to propose to stakeholders. Each phase is independently shippable and gives clear value before the next phase starts.

Phase 1 — Live Integration ~4 weeks

Tiers 1 + 2 + 10. Live order placement (PDC and PDR via single ProductCode + LPA Key/CaseFile), automated report receipt back into JaroDesk, status updates visible in JaroDesk notes + Teams, hardened against the known production gotchas (multi-shape error coalesce, SenderID validation, retry policies).

Decisions needed: none beyond what we already have. Independent of any business workflow design.

Phase 2 — Operational + Financial +6–8 weeks

Tiers 3 + 5. Adds Opteon-initiated cancellation lifecycle and fee adjustment approval workflow. These are the two most-frequently-used post-MVP features in similar AMC integrations.

Decisions needed: two design conversations — cancellation trigger mechanism and fee approval thresholds/authority.

Phase 3 — Roadmap Items Open-ended

Tiers 4, 6, 7, 8, 9. Revisions, due date negotiation, QC, UAD compliance, edge-case message types. Each has its own design dependency and stakeholder owner.

Decisions needed: a workflow design conversation per tier, scoped individually.

Three Decision Points to Surface in the Meeting

  1. How urgent is going live with Tier 1? If urgent, Phase 1 ships in ~4 weeks and Phase 2 follows. If not urgent, the broader scope can be planned together.
  2. Where's the "good enough" line? Tier 1 = "we can place orders and get reports." Tier 2 = visibility but no new functional features. Tier 3+ = operational features. Where do we declare the integration complete?
  3. Who owns the workflow design conversations for Phase 2 and Phase 3? If those owners aren't identified now, those tiers don't move regardless of dev capacity. The bottleneck will be people, not code.
